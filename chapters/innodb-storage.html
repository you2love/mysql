1. 数据页存储

1.1 数据页概述

InnoDB 是 MySQL 默认的存储引擎，其数据存储的基本单位是数据页（Page）。每个数据页默认大小为 16KB，是 InnoDB 与磁盘交互的最小单位。

-- InnoDB 数据页基本参数：
-- - 页大小: 16KB (可通过 innodb_page_size 配置)
-- - 页类型: 数据页、索引页、Undo页、系统页等
-- - 磁盘IO: 每次读写以页为单位
-- - 缓存: 页数据缓存在 Buffer Pool 中

-- 查看数据页大小：
SHOW VARIABLES LIKE 'innodb_page_size';

1.3 数据页查找过程

-- 数据页内查找记录的过程：

-- 1. 通过 B+ Tree 定位到目标数据页
-- 2. 在数据页内使用 Page Directory 进行二分查找
-- 3. 定位到记录所在的 Slot
-- 4. 顺序查找定位精确记录
-- 5. 返回记录（或通过主键回表获取完整数据）

2. ibd 文件存储

2.1 ibd 文件概述

ibd 文件是 InnoDB 存储引擎的表空间文件，存储了表数据和索引。每个 InnoDB 表都有一个独立的 ibd 文件（独立表空间模式）。

-- ibd 文件结构：
-- 每个 ibd 文件由多个 16KB 的数据页组成
-- 页类型：
--   - FIL_PAGE_INDEX: 索引页（ B+ Tree 叶子节点）
--   - FIL_PAGE_TYPE_SYS: 系统页
--   - FIL_PAGE_TYPE_TRX: 事务页
--   - FIL_PAGE_TYPE_BLOB: BLOB 页

-- 查看表对应的 ibd 文件：
SHOW VARIABLES LIKE 'innodb_file_per_table';
-- ON: 每个表独立 ibd 文件
-- OFF: 所有表共享 system tablespace

2.3 区(Extent)与段(Segment)

InnoDB 使用区(Extent)作为空间分配单位，每个区包含 64 个连续的数据页（1MB）。

-- 区(Extent)结构：
-- 1 区 = 64 页 = 1MB

-- 段(Segment)结构：
-- 每个索引包含两个段：
--   - Leaf Node Segment: 存储所有数据行
--   - Non-Leaf Node Segment: 存储索引键

-- 区分配策略：
-- 1. 碎片页：前 32 页按需分配
-- 2. 完整区：32 页后整区分配

2.4 表空间Tablespace

-- 1. 系统表空间 (System Tablespace)
--    文件：ibdata1, ibdata2, ...
--    存储：undo log, change buffer, doublewrite buffer

-- 2. 独立表空间 (File-Per-Table)
--    文件：table_name.ibd
--    推荐：innodb_file_per_table = ON

-- 3. 通用表空间 (General Tablespace)
CREATE TABLESPACE ts1 ADD DATAFILE 'ts1.ibd';
CREATE TABLE t1 (id INT) TABLESPACE ts1;

-- ibd 文件读写优化：

-- 1. Buffer Pool 调优
SET GLOBAL innodb_buffer_pool_size = 4 * 1024 * 1024 * 1024;
-- 建议设置为可用内存的 50-70%

-- 2. 刷新策略
SET GLOBAL innodb_flush_log_at_trx_commit = 1;
-- 1: 每次事务提交刷新日志（最安全）

-- 查看状态：
SHOW ENGINE INNODB STATUS;

2.6 ibd 文件空间回收

-- 空间回收方法：

-- 1. 重建表回收空间
OPTIMIZE TABLE table_name;

-- 2. 重建表
ALTER TABLE table_name ENGINE=InnoDB;

-- 查看表空间使用：
SELECT 
    table_name,
    round(data_length/1024/1024, 2) as data_mb
